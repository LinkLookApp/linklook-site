LinkLook — Default Browser Activation Flow
Version: v2.0
Date: 2026-03-16
Origin: Address-now hardening pass, item #2; updated with iOS 18.2 API
References: Entry_Path_Visibility.txt, Start_Page_Design.txt,

================================================================================
1. PURPOSE
================================================================================

LinkLook only works when it is the default browser. If the user installs
LinkLook but does not set it as default, they get zero protection. This
document defines the activation flow that guides users through setup and
keeps them aware of their default-browser status.

================================================================================
2. CORE PROBLEM
================================================================================

iOS does not let apps set themselves as default browser programmatically.
The user must go to iOS Settings → LinkLook → Default Browser App and
select LinkLook manually. This is a conscious Apple design choice.

LinkLook must make this step as easy as possible without being pushy.

================================================================================
3. FIRST-LAUNCH ONBOARDING (3 screens)
================================================================================

The onboarding flow appears once, on first launch only. It cannot be
skipped entirely — the user must tap through all three screens. Each
screen has a single "Continue" button at the bottom.

--------------------------------------------------------------------------------
3.1 SCREEN 1 — WELCOME
--------------------------------------------------------------------------------

  Title:    "LinkLook checks links before you open them"
  Subtext:  "When someone sends you a link, LinkLook looks at it first
             and tells you if it's safe."
  Visual:   Simple illustration or icon (shield + magnifying glass)
  Button:   "Continue"

--------------------------------------------------------------------------------
3.2 SCREEN 2 — SET AS DEFAULT BROWSER
--------------------------------------------------------------------------------

  Title:    "Set LinkLook as your default browser"
  Subtext:  "Use the Apple Settings button to make LinkLook the
             default browser."

  Button:   "⚙ Apple Settings"  (gearshape.fill icon + text)
            → calls UIApplication.shared.open(URL(string:
              UIApplication.openSettingsURLString)!)
            → opens the LinkLook section in iOS Settings
            → iOS shows the correct path regardless of version

  No step-by-step instructions — the exact path in Settings differs
  across iOS versions. The deep link takes the user to the right place.

  Below the button:
    "Already done? Tap Continue."
    [Continue] (secondary style, less prominent)

  Note: this screen does NOT block the user. If they tap "Continue"
  without setting default, the app still opens — but the start page
  will show a reminder (see section 5).

--------------------------------------------------------------------------------
3.3 SCREEN 3 — COVERAGE EXPLANATION
--------------------------------------------------------------------------------

  Title:    "What LinkLook checks"
  Body:     The in-app explanation text from Coverage_Limits_Wording.txt
            (block 2.1):

            "LinkLook checks web links that reach it as your default
             browser — like links you tap in Messages, Mail, or WhatsApp,
             and QR codes that open as web addresses.

             Some links bypass any browser. Apps that open links inside
             their own viewer, or links that go directly to another app,
             won't pass through LinkLook. You can always paste any link
             into LinkLook to check it manually."

  Button:   "Got it" → dismisses onboarding, shows start page

================================================================================
4. DEFAULT BROWSER STATUS DETECTION
================================================================================

Since iOS 18.2 / iPadOS 18.2, Apple provides a direct API for browser
apps to check their default status. For older iOS versions, LinkLook
falls back to a heuristic. Both paths feed into a unified
DefaultBrowserStatus model.

--------------------------------------------------------------------------------
4.1 iOS 18.2+ — UIApplication.isDefault(.browser)
--------------------------------------------------------------------------------

  API:      UIApplication.shared.isDefault(.browser) → Bool
  Source:   Apple documentation for default-browser setup (iOS 18.2+)

  WHEN TO CHECK:
    - On app launch (application didFinishLaunching)
    - Every time the app becomes active (sceneDidBecomeActive /
      .scenePhase == .active)

  HOW IT WORKS:
    - Call isDefault(.browser) and compare against the stored last-known
      state in UserDefaults (key: lastKnownDefaultBrowserState).
    - The API is a status check, NOT a subscription. There is no
      callback or notification when the user changes the setting.
      Polling on didBecomeActive is the recommended approach.

  STATE TRANSITIONS:

    true → false (user switched away from LinkLook):
      Show a calm, non-blocking banner on the start page:
        "LinkLook is not your default browser."
        Bold: "To stay protected against unsafe links, use the
        Apple Settings button to make LinkLook the default browser."
        Full-width green "Apple Settings" button (same style as onboarding).
      - Banner appears at most ONCE per app-foreground cycle.
      - No modals, no alerts, no interstitials. Apple constrains
        this API to prevent nagging behavior. Respect that.

    false → true (user set LinkLook as default):
      - Update the Settings screen checkmark.
      - Dismiss any active reminder banner.
      - Optionally show a brief confirmation toast:
        "LinkLook is your default browser ✓"

    true → true / false → false (no change):
      - No action needed.

  WHAT THE API DOES NOT TELL YOU:
    - Which other browser is the current default.
    - The exact moment the user changed the setting (only detectable
      on next didBecomeActive).

  PRIVACY / APP REVIEW:
    - The API is designed for browser apps. Using it is expected behavior.
    - Do not call it in rapid loops or from background tasks.

--------------------------------------------------------------------------------
4.2 PRE-iOS 18.2 — HEURISTIC FALLBACK
--------------------------------------------------------------------------------

  On iOS versions before 18.2, isDefault(.browser) is unavailable.
  LinkLook falls back to a handoff heuristic:

  HEURISTIC:
    LinkLook tracks whether it has ever received a URL via the default-
    browser handoff path (http/https scheme in LinkLookApp.onOpenURL).

    - If at least one URL has been received via handoff → status = ACTIVE
    - If no URL has ever been received via handoff → status = UNKNOWN
    - On first launch before any interaction → status = NOT_SET

    The status is stored locally:
      hasReceivedDefaultBrowserHandoff: Bool (UserDefaults)

  LIMITATIONS:
    This heuristic CANNOT detect if the user later changes the default
    browser away from LinkLook. It only knows LinkLook was default at
    some point.

  FIRST-LAUNCH MESSAGE (pre-18.2 only):
    Show a one-time informational message the first time the app opens
    after install or update:

      "Make sure LinkLook is your default browser to stay protected.
       For the best experience, update to the latest iOS version."

    Show once, then never again. Store flag:
      hasShownPreiOS182DefaultMessage: Bool (UserDefaults)

    This message is different from the iOS 18.2+ banner because:
    - We cannot know for certain whether LinkLook is default.
    - We want to gently encourage iOS updates for better detection.

--------------------------------------------------------------------------------
4.3 RUNTIME BRANCHING
--------------------------------------------------------------------------------

  Use `if #available(iOS 18.2, *)` to branch between the two paths:

    if #available(iOS 18.2, *) {
        // Use UIApplication.shared.isDefault(.browser)
        let isDefault = UIApplication.shared.isDefault(.browser)
        // Compare with stored state, update UI accordingly
    } else {
        // Fall back to handoff heuristic
        // Use hasReceivedDefaultBrowserHandoff flag
    }

  Both paths produce a unified DefaultBrowserStatus enum:

    enum DefaultBrowserStatus {
        case isDefault          // Confirmed default (API or handoff received)
        case notDefault         // Confirmed NOT default (API only, iOS 18.2+)
        case unknown            // Cannot determine (pre-18.2, no handoff yet)
        case notSet             // First launch, pre-onboarding
    }

  The rest of the app (start page, settings, onboarding, banners) reads
  DefaultBrowserStatus without caring which detection method produced it.

5. START PAGE REMINDER (not-default state)
================================================================================

The start page reminder adapts based on the detection method:

  iOS 18.2+ (confirmed not-default):
  ┌──────────────────────────────────────────────────────────────────┐
  │  LinkLook is not your default browser.                           │
  │                                                                  │
  │  **To stay protected against unsafe links, use the              │
  │  Apple Settings button to make LinkLook the default             │
  │  browser.**                                                      │
  │                                                                  │
  │  ┌──────────────────────────────────────────────────────────┐    │
  │  │              Apple Settings                              │    │
  │  └──────────────────────────────────────────────────────────┘    │
  │   (green filled button, same style as onboarding)          [×]  │
  └──────────────────────────────────────────────────────────────────┘

  Pre-iOS 18.2 (uncertain, first launch / no handoff received):
  ┌──────────────────────────────────────────────────────────────────┐
  │  LinkLook is not your default browser.                           │
  │                                                                  │
  │  **To stay protected against unsafe links, use the              │
  │  Apple Settings button to make LinkLook the default             │
  │  browser.**                                                      │
  │                                                                  │
  │  ┌──────────────────────────────────────────────────────────┐    │
  │  │              Apple Settings                              │    │
  │  └──────────────────────────────────────────────────────────┘    │
  │   (green filled button, same style as onboarding)          [×]  │
  └──────────────────────────────────────────────────────────────────┘

  Both variants:
  - Card style: same as primary action card, subtle amber/warm background
  - "Apple Settings" button: full-width green filled button
    (Theme.primaryGreen background, white semibold text, rounded 12pt
    corners) — identical to the onboarding screen 2 button. No gear icon.
  - Deep-links to LinkLook's iOS Settings page via
    UIApplication.openSettingsURLString — iOS shows the correct
    path regardless of version. No hardcoded navigation instructions.
  - Disappears when ANY of these conditions is met:
    (a) DefaultBrowserStatus becomes .isDefault
    (b) History has entries (any link checked, regardless of entry method)
    (c) User dismisses manually (× button) — per-session only on pre-18.2,
        persistent dismiss on iOS 18.2+ since we can re-show if status
        changes again
  - On iOS 18.2+: if the user dismisses and the status later changes
    back to notDefault, show the banner again (max once per foreground)

================================================================================
6. SETTINGS SCREEN STATUS ROW
================================================================================

The Settings tab gets a new row at the top of the settings list:

  Default browser status:

  If .isDefault:
    "Default browser    LinkLook ✓"
    (green checkmark, no action needed)

  If .notDefault (iOS 18.2+ only):
    "Default browser    Not set"
    (tappable → opens iOS Settings)
    Subtitle: "Tap to open Settings and select LinkLook"

  If .unknown (pre-18.2):
    "Default browser    Check Settings"
    (tappable → opens iOS Settings)
    Subtitle: "Make sure LinkLook is selected"

================================================================================
7. FAMILY INSTALLER SCENARIO
================================================================================

For the primary use case (adult child installs LinkLook for elderly
parent), the onboarding serves double duty:

  - The installer walks through screens 1-3 during setup
  - Screen 2 ensures the installer sets the default browser
  - Screen 3 gives the installer the honest coverage explanation

  The parent (primary user) then sees:
  - The start page with status confirmation (if set correctly)
  - Or the gentle reminder (if the installer forgot)

  On iOS 18.2+: the installer can verify immediately via the Settings
  screen that the green checkmark appears. On older iOS, they rely on
  the onboarding flow and can test by tapping a link in Messages.

  Future enhancement: the device-level profile settings (PROTECTED /
  STANDARD) from Architecture.txt should integrate with this flow,
  but that is out of scope for this document.

================================================================================
8. MINIMUM DEPLOYMENT TARGET CONSIDERATIONS
================================================================================

  - LinkLook's minimum deployment target should be iOS 17.0 or later
    (to be confirmed per project requirements).
  - The isDefault(.browser) API requires iOS 18.2. This means a
    significant portion of users (on iOS 17.x–18.1) will use the
    heuristic fallback for at least one year after launch.
  - As iOS 18.2+ adoption grows, the heuristic code path will
    naturally phase out. Do not remove it until the minimum deployment
    target is raised to iOS 18.2.

================================================================================
9. METRICS TO TRACK LATER
================================================================================

These are not MVP requirements, but should be planned for:

  - Install → default browser activation rate
    (percentage of installs that reach .isDefault state)
  - Time from install to first confirmed default status
  - Default browser retention (now possible on iOS 18.2+ via the API)
  - Rate of true→false state changes (users switching away)
  - Onboarding completion rate (if analytics are added)
  - iOS version distribution (to track isDefault API availability)
