Security Warning User Experience
Generated: 2026-03-08

Purpose
Define how LinkLook should communicate SAFE, SUSPICIOUS, and BLOCK decisions without confusing or overwhelming the user.

Principles
- plain language first
- explain the reason briefly
- avoid panic wording
- let the user understand what happened
- reserve hard blocking for high-confidence risk

Example states
SAFE
- Message: This link looks safe.
- Action: Page loads in the minimal browser after verdict.
- Display mode (MDL-109): "always show result" (default) shows the safe screen
  briefly for reassurance; "auto-open safe links" skips the screen and loads
  directly. PROTECTED profile enforces always show result.
- Open elsewhere button available to hand off to preferred browser (MDL-104).

SUSPICIOUS
- Message: This link has warning signs.
- Explain one or two reasons such as new domain, hidden destination, or brand imitation.
- Action: Let user stop or continue with friction.

BLOCK
- Message: This link appears dangerous and was blocked.
- Explain the strongest reason available.
- Action: Default to stop.

Browser preference
The user chooses which browser safe links open in via Settings. Safari is the
default redirect browser out of the box. LinkLook is a safety-driven default
browser — it is never a redirect target itself. The preference list only shows
browsers that are installed on the device. If the chosen browser is no longer
available, LinkLook falls back to Safari.

Special cases
- QR-origin links
- shortener links
- Universal Link / app-opening behavior
- repeat attempts on the same high-risk destination

Measurement
Track whether warnings reduce risky opens without creating excessive false alarms.
