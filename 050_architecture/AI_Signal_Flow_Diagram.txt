LinkLook — AI Signal Flow Diagram
Status: active
Date: 2026-03-13

Purpose
Show how AI signals flow through the LinkLook check pipeline. This diagram
is the reference for understanding where AI fits in the architecture.

================================================================================
TERMINOLOGY (use consistently across all docs)
================================================================================

  Term                  Meaning
  ──────────────────────────────────────────────────────────────────────────
  AI signal model       Generic product term for any on-device AI model
  URL signal model      Link classifier — scores URLs for phishing risk
  Message signal model  Text classifier — scores message text for scam patterns
  Core ML               Deployment/runtime technology (Apple on-device ML)
  Foundation Models     Apple's optional on-device explanation layer (future)
  Backend LLM           Optional future remote analysis layer (deferred)

  Avoid: "SLM" as a product term (use only when discussing model size class).
  Avoid: "Claude" as an architectural component (use "backend LLM" if needed).

================================================================================
KEY INVARIANTS
================================================================================

1. The deterministic path always runs. AI is optional.
2. AI models produce signals, not verdicts.
3. The combiner is deterministic, not ML.
4. AI can escalate, never downgrade, never override BLOCK.
5. "The AI said so" is never shown to the user — pattern-specific messages only.
6. The base product works correctly with AI disabled.
7. Backend LLM is not a required runtime stage.
