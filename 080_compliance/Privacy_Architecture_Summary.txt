LINKLOOK — PRIVACY ARCHITECTURE SUMMARY
Version: 1.2
Date: 2026-03-10
Origin: ChatGPT, updated to align with Canonical Master Decision Log 2026-03-10

Privacy promise
LinkLook protects links without building a browsing-surveillance product.

Privacy-first principle (MDL-084)
LinkLook follows a privacy-first approach by default. It uses only the minimum
data necessary for link-safety, avoids hidden behavioral profiling, and remains
clear and transparent about what data is processed and why. Where user-specific
trust or preference data exists, it stays on-device where possible.
Privacy-impacting exceptions are only allowed when clearly necessary for
security, proportionate, protected, and transparent.

Straightforward, security-driven browser design
- LinkLook is a straightforward, security-driven browser for iOS.
- The browser receives the destination URL before navigation.
- The destination is evaluated for safety signals.
- The canonical outcome model is: allow (transparent), inform, warn, block.
- Forward navigation remains inside LinkLook, not as a separate mode or handoff product.
- LinkLook does not act as a VPN or general traffic proxy in v1.

Two-data-flow separation (MDL-100, MDL-108)
Personal trust data (allow/block choices, preferences) stays on-device and
never leaves. Threat signal data (domain-level observations, not user-level
data) may be submitted anonymously, aggregated centrally, and fed back into a
shared blocklist — but only when the user has explicitly opted in (MDL-108).
The opt-in toggle is OFF by default and lives in the in-app settings screen.
When the toggle is off, no outbound signal data leaves the device at all.
Differential privacy and k-anonymity are specified as the privacy mechanisms.
A domain only enters the global shared list after 1,000+ distinct user reports.

Data-minimisation principles
- LinkLook stores a local check history by default (URL, verdict, reason code,
  timestamp) to support Proof of Activity and user trust (MDL-106). This is not
  a general browsing history — it records only LinkLook's own safety checks.
  The user can clear or disable it from in-app settings. History data is
  personal trust data (MDL-100) and never leaves the device.
- do not collect personal messages
- minimise retained link-analysis data
- separate technical threat-intelligence storage from user identity

Proactive reclassification notification (MDL-101)
If a link that LinkLook previously cleared as safe is later identified as
malicious, affected users receive a proactive plain-English notification with
actionable steps. This is an expression of the honesty principle, not a
privacy risk — the notification is local and does not expose browsing data.

Network Extension note
Network Extension is planned for V3. If added, privacy boundaries must remain
explicit. The extension should process only the minimum signals needed for the
feature and must not undermine the core privacy claim.
