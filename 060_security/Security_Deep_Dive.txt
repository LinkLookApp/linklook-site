1. System Architecture Overview

Core Flow:
1. Browser Interception Layer
2. Deterministic Checks
3. Sandboxed Docking Environment
4. SLM (future primary classifier)
5. LLM (reasoning & audit)
6. Structured Event Emission (ADR-003)
7. Optional Claims Evidence Logging (ADR-004)

All components are modular and replaceable.




2. Docking Architecture (Sandboxed Fetch)

- Headless isolated container per request
- No execution in user browser context
- CPU/memory/time limits enforced
- Network egress restricted
- Disposable instances (no state persistence)

5. Privacy-by-Design Enforcement Controls

- No IP address persistence
- Query parameters stripped or hashed
- No raw HTML stored by default
- Short-lived debug storage (<=14 days)
- Evidence vault retention 7–30 days
- Automated deletion jobs
- Schema validation in CI




6. Integrity & Tamper Evidence

- Append-only log store
- Event signing (server-side key)
- Optional per-device hash chaining
- Access logging for all evidence reads
- Key rotation policy for encryption keys




7. Model Strategy & Learning Pipeline

- Deterministic → SLM (target >=80% coverage) → LLM audit
- Structured dataset accumulation from v1
- No live self-modifying production models
- Offline retraining with version control
- Rollback capability for every model version




8. Observability & Metrics

Key Metrics:
- Interception latency (p80, p95)
- False positive rate
- Escalation rate (future SLM metric)
- User back-out rate
- Uptime

All metrics aggregated without reconstructing browsing trails.
